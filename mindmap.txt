期間横断シミュレーション ウィザード操作マニュアル

期間横断シミュレーション
├─ 対象
│  ├─ The競馬 EV分析 スマホ版
│  ├─ 初心者向け
│  ├─ JRA・NAR対応
│  ├─ 過去データを使う検証機能
│  ├─ 実際のお金は動かない
│  └─ 実際の投票や購入は行わない
│
├─ 期間横断シミュレーションとは
│  ├─ 開始日から終了日までの開催を調べる
│  ├─ 現在の予想設定を使用する
│  ├─ 現在の予算設定を使用する
│  ├─ 条件に基づいて買い目を作る
│  ├─ 確定済みレースの払戻を取得する
│  ├─ 収支を計算する
│  ├─ 開催日別に集計する
│  ├─ 開催場別に集計する
│  └─ 全期間トータルを集計する
│
├─ 基本操作
│  ├─ 1. ウィザードを開く
│  │  ├─ 画面上部の「♞ シミュレーション」を開く
│  │  ├─ 画面を下へスクロールする
│  │  ├─ 「期間×開催場 横断シミュレーション」を探す
│  │  └─ スマホでは項目が縦に表示される
│  │
│  ├─ 2. 区分を選ぶ
│  │  ├─ JRA
│  │  │  ├─ 中央競馬
│  │  │  ├─ 東京
│  │  │  ├─ 中山
│  │  │  ├─ 京都
│  │  │  └─ 阪神
│  │  └─ NAR
│  │     ├─ 地方競馬
│  │     ├─ 盛岡
│  │     ├─ 水沢
│  │     ├─ 大井
│  │     ├─ 園田
│  │     └─ 高知
│  │
│  ├─ 3. 期間を設定する
│  │  ├─ 開始日
│  │  │  └─ 調べたい最初の日
│  │  ├─ 終了日
│  │  │  └─ 調べたい最後の日
│  │  └─ 注意事項
│  │     ├─ 終了日は開始日より後にする
│  │     ├─ 同じ日を指定すると1日分だけ調べる
│  │     ├─ 最初は3日から7日程度で試す
│  │     └─ 慣れてから期間を広げる
│  │        ├─ 1か月
│  │        ├─ 3か月
│  │        └─ 1年
│  │
│  ├─ 4. 開催場を選ぶ
│  │  ├─ 特定場だけ調べる
│  │  │  └─ 対象の競馬場にチェックを付ける
│  │  ├─ 何も選ばない
│  │  │  └─ 実際に開催された全競馬場が対象になる
│  │  ├─ 操作ボタン
│  │  │  ├─ 全選択
│  │  │  └─ 全解除
│  │  └─ 使い分け
│  │     ├─ 全体傾向を見る
│  │     │  └─ 何も選ばない
│  │     ├─ 特定場だけ検証する
│  │     │  └─ その競馬場だけ選ぶ
│  │     └─ 複数場を比較する
│  │        └─ 比較したい競馬場を複数選ぶ
│  │
│  ├─ 5. シミュレーションを実行する
│  │  ├─ 実行前に確認する
│  │  │  ├─ 区分
│  │  │  ├─ 開始日
│  │  │  ├─ 終了日
│  │  │  ├─ 開催場
│  │  │  ├─ 予想設定
│  │  │  └─ 予算設定
│  │  ├─ 実行
│  │  │  └─ 「期間横断シミュレーション実行」を押す
│  │  ├─ 高速モード
│  │  │  ├─ 長期間の処理に向いている
│  │  │  ├─ 通信回数を減らす
│  │  │  ├─ 画面更新を減らす
│  │  │  ├─ 大量レースの処理を軽くする
│  │  │  ├─ 計算方法は通常モードと同じ
│  │  │  └─ 出力形式は通常モードと同じ
│  │  ├─ 地方移動EV補正
│  │  │  └─ 必要に応じて使用する
│  │  └─ 実行中の注意
│  │     ├─ 画面を閉じない
│  │     ├─ ブラウザをバックグラウンドへ移さない
│  │     ├─ 処理が止まる場合がある
│  │     └─ 処理が遅くなる場合がある
│  │
│  └─ 6. 結果を確認する
│     ├─ 開催日別集計
│     ├─ 開催場別集計
│     ├─ 全期間トータル
│     ├─ 投資
│     │  └─ 買ったと想定される金額
│     ├─ 払戻
│     │  └─ 的中した払戻総額
│     ├─ 収支
│     │  └─ 払戻－投資
│     └─ ROI
│        ├─ 計算式
│        │  └─ 払戻÷投資×100
│        ├─ 100%より上
│        │  └─ プラス
│        ├─ 100%
│        │  └─ 収支ゼロ
│        └─ 100%より下
│           └─ マイナス
│
├─ 中断と再開
│  ├─ 中断
│  │  ├─ 「⏹ 中断」を押す
│  │  ├─ 現在の進行位置を残す
│  │  └─ 処理を停止する
│  ├─ 続きから再開
│  │  ├─ 同じページを開く
│  │  ├─ 「▶ 続きから再開」を押す
│  │  └─ 未処理部分から再開する
│  └─ 長期間処理の保存
│     ├─ 「期間横断JSON出力」を使う
│     └─ 途中結果を保存する
│
├─ 初心者向けおすすめ設定
│  ├─ 期間
│  │  └─ 3日から7日
│  ├─ 区分
│  │  └─ JRAかNARの片方
│  ├─ 開催場
│  │  └─ 1場だけ
│  ├─ 高速モード
│  │  └─ ON
│  ├─ 1点額
│  │  └─ 100円
│  ├─ 日次上限
│  │  └─ 1,800円
│  └─ 確認項目
│     ├─ 投資
│     ├─ 払戻
│     ├─ 収支
│     └─ ROI
│
├─ 困ったとき
│  ├─ 実行ボタンを押しても進まない
│  │  ├─ 通信環境を確認する
│  │  ├─ ページを再読み込みする
│  │  └─ 期間を3日程度へ短くする
│  │
│  ├─ 開催場が表示されない
│  │  ├─ 区分を先に設定する
│  │  ├─ 期間を先に設定する
│  │  └─ 開催のない日だけを指定していないか確認する
│  │
│  ├─ 結果が0件になる
│  │  ├─ 未確定レースは精算対象外
│  │  ├─ 過去の確定済み開催日を指定する
│  │  ├─ 予想条件が厳しすぎないか確認する
│  │  └─ 条件によっては購入対象が0Rになる
│  │
│  ├─ 予算が余っているのに購入数が少ない
│  │  ├─ 日次上限は必ず使い切る金額ではない
│  │  ├─ 旨み条件を通過した3点セットだけを採用する
│  │  ├─ 条件未達レースは見送る
│  │  └─ 予算が残っていても無理に買わない
│  │
│  └─ 1年分を一度に調べたい
│     ├─ 高速モードをONにする
│     ├─ 途中でJSON保存する
│     ├─ 数か月ごとに分ける
│     └─ 分割した方が安定する
│
├─ 最後の確認チェックリスト
│  ├─ JRA／NARを正しく選んだ
│  ├─ 開始日と終了日が正しい
│  ├─ 開催場の選択方法を確認した
│  ├─ 1日の購入上限を確認した
│  ├─ 1点額を確認した
│  ├─ 短い期間で試した
│  ├─ 慣れてから長期間へ広げた
│  ├─ 全期間トータルの投資を確認した
│  ├─ 全期間トータルの払戻を確認した
│  └─ 全期間トータルのROIを確認した
│
├─ 運用方針
│  ├─ 最初は小さく試す
│  ├─ 結果を確認してから期間を広げる
│  ├─ 予算を無理に使い切らない
│  ├─ 条件未達レースは見送る
│  ├─ 長期間処理は保存しながら進める
│  └─ 投資額だけでなくROIと収支も確認する
│
